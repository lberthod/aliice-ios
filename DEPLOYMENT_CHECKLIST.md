# ✅ Pre-Launch Checklist for App Store

## PHASE 1: BLOCKERS (Fix These First!)

### [ ] 1. Get Apple Developer Account + Team ID
```bash
# Go to developer.apple.com
# Enroll in Apple Developer Program ($99/year)
# Save your Team ID (10 characters, e.g., "ABC123XYZ9")
```

### [ ] 2. Create App Icons
Use one of these free tools:
- https://www.appicon.co (upload 1024x1024, auto-generates all sizes)
- https://www.iconkitchen.app/ (online, free)
- Figma (design tool)

**Required sizes:**
- 1024×1024 (marketing/store)
- 180×180 (iPhone spotlight)
- 120×120 (iPhone app)
- 80×80 (iPhone spotlight small)
- 40×40 (iPhone app small)
- Plus iPad, watch, CarPlay variants

**Where to put them:**
```
AliiceIOS/
  Assets.xcassets/
    AppIcon.appiconset/
      (all the generated PNG files here)
      Contents.json (auto-generated by tool)
```

### [ ] 3. Create LaunchScreen.storyboard
This is what users see when the app boots (1-2 seconds)

**Steps:**
1. Xcode → File → New → Storyboard
2. Name it `LaunchScreen.storyboard`
3. Place View Controller in center
4. Add:
   - UIImageView: "🇹🇭" emoji or logo (size: 100×100)
   - UILabel: "Aliice" (font: bold 36)
   - UILabel: "Learn Thai" (font: light 16)
5. Center all vertically

Alternative: Copy existing LaunchScreen structure

### [ ] 4. Update project.yml with Team ID
```yaml
# project.yml
settings:
  base:
    DEVELOPMENT_TEAM: "YOUR_TEAM_ID_HERE"  # 10-char code from Step 1

targets:
  AliiceIOS:
    settings:
      base:
        PRODUCT_BUNDLE_IDENTIFIER: com.yourcompany.aliice
```

Then regenerate: `xcodegen generate`

### [ ] 5. Create PrivacyInfo.xcprivacy
**File:** `AliiceIOS/PrivacyInfo.xcprivacy`

```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
	<key>NSPrivacyTracking</key>
	<false/>
	<key>NSPrivacyTrackingDomains</key>
	<array/>
	<key>NSPrivacyAccessedAPITypes</key>
	<array>
		<dict>
			<key>NSPrivacyAccessedAPIType</key>
			<string>NSPrivacyAccessedAPISpeechRecognitionUsage</string>
			<key>NSPrivacyAccessedAPITypeReasons</key>
			<array>
				<string>1EF6</string>
			</array>
		</dict>
	</array>
</dict>
</plist>
```

Add to project: Xcode → Add Files → PrivacyInfo.xcprivacy → Check "Copy items"

## PHASE 2: CRITICAL (Fix After Phase 1)

### [ ] 6. Create Privacy Policy
**Template:** Use https://www.privacypolicygenerator.info/

Key points for Aliice:
- App collects game progress (local storage only)
- No personal data transmitted
- No ads or tracking
- GDPR compliant (EU)

**Example:**
```
Aliice - Privacy Policy

We are committed to protecting your privacy.

Data Collection:
- Game progress (scores, categories) stored locally on your device
- No server transmission
- No analytics or tracking
- No personal information required

Data Storage:
- All data stored in iOS UserDefaults
- Deleted when app is uninstalled

Contact: [your email]
```

**Host it:** 
- GitHub Pages (free)
- Vercel (free)
- Firebase Hosting (free tier)
- Or your website

**Save URL:** `https://yourdomain.com/privacy`

### [ ] 7. Create Terms of Service
**Template:** Use iubenda.com or termly.io (free tiers available)

Key points:
- Educational app for learning Thai
- No warranty
- License to use only for personal use
- Apple's standard ToS apply

**Save URL:** `https://yourdomain.com/terms`

### [ ] 8. Fix Bundle ID
```yaml
# project.yml - change:
bundleIdPrefix: com.berthod          # your reverse domain
# to:
bundleIdPrefix: com.aliice           # or use your company
```

Then: `xcodegen generate`

### [ ] 9. Update Version Number
```yaml
# project.yml
info:
  properties:
    CFBundleShortVersionString: "1.0.0"  # User-facing version
    CFBundleVersion: "1"                 # Build number (int only)
```

Then: `xcodegen generate`

## PHASE 4: BUILD & TEST

### [ ] 14. Build for Simulator
```bash
cd /Users/berthod/Desktop/AliiceIOS
DEVELOPER_DIR=/Applications/Xcode.app/Contents/Developer \
xcodebuild -project AliiceIOS.xcodeproj \
  -scheme AliiceIOS \
  -destination "generic/platform=iOS Simulator" \
  build
```

### [ ] 15. Test on Multiple Simulators
```bash
# Test iPhone SE (small), iPhone 15 Pro (large), iPhone 15 Pro Max (huge)
# Xcode → Product → Destination → Select each
# Run through full gameplay:
# - Play one category completely
# - Unlock next category
# - Check profile & badges
# - Kill app, reopen → check persistence
# - Tap TTS multiple times → check 3s cooldown works
```

### [ ] 16. Test Accessibility
```
Settings → Accessibility:
- Large Text Size (max)
- VoiceOver (should navigate fine)
- Dark Mode (should be readable)
```

### [ ] 17. Memory Leak Check (Instruments)
```bash
xcodebuild build-for-testing
Xcode → Product → Scheme → Edit Scheme
→ Run → Diagnostics → Check "Malloc Stack Logging"
→ Run app, play 50+ rounds
→ Xcode → Debug → Allocations/Leaks
```

Ensure no growing memory usage or leaked objects.

## PHASE 5: SUBMIT TO APP STORE

### [ ] 18. Create App in App Store Connect
1. Visit https://appstoreconnect.apple.com
2. Sign in with Apple ID
3. Apps → New App
4. Platform: iOS
5. Name: "Aliice"
6. Bundle ID: com.yourcompany.aliice
7. SKU: aliice-thai (internal ID, any string)
8. User Access: Full Access
9. Create

### [ ] 19. Fill in App Information
In App Store Connect:
- **Version Release:** Select "Manually Release"
- **Description:** Paste from Phase 3
- **Keywords:** Paste from Phase 3
- **Support URL:** Paste from Phase 3
- **Privacy Policy:** Paste from Phase 3
- **Category:** Education
- **Rating:** Fill out IARC (auto-detect as 4+)

### [ ] 20. Upload Icons & Screenshots
- Icons: Upload 1024×1024 PNG
- Screenshots: Upload 1170×2532 PNGs
- Preview Images (optional)

### [ ] 21. Build Archive & Upload
```bash
# Clean previous builds
xcodebuild -project AliiceIOS.xcodeproj \
  -scheme AliiceIOS \
  clean

# Create Release Archive
xcodebuild -project AliiceIOS.xcodeproj \
  -scheme AliiceIOS \
  -configuration Release \
  -sdk iphoneos \
  -archivePath build/Aliice.xcarchive \
  archive

# Upload (Xcode GUI):
# Xcode → Window → Organizer
# Select Archive → Distribute App
# → TestFlight or App Store Connect
# → Validate & Submit
```

### [ ] 22. Review & Submit
- Review all information
- Check "Comply with Export Compliance"
- Submit for Review

**Status:** "Waiting for Review" (24-48 hours)

### [ ] 23. Monitor Review Status
- Check App Store Connect daily
- If rejected: Fix issues & resubmit
- If approved: 🎉 Live on App Store!

## COMMON REJECTION REASONS

- ❌ Missing Privacy Policy → Add Privacy Policy URL
- ❌ Crash on launch → Fix LaunchScreen storyboard
- ❌ No icons → Add all icon sizes
- ❌ Unclear app purpose → Improve description
- ❌ Poor quality → Test more thoroughly
